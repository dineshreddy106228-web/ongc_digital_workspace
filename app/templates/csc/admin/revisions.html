Type Classification Workbook Import

Upload one `.xlsx` workbook from the Type Classification Exercise. The Secretary Workbench will match each sheet to a published specification, create or reopen the existing workflow draft, and prefill parameter type/test-procedure updates plus the Justification tab.

One workbook = one subset submission file. Blank sheets/rows are skipped automatically.

Expected Workbook Format

  • One sheet per specification.
  • Required matching columns: Spec Number, Material Code, S. No., Type.
  • Imported parameter fields: Type, Test Procedure Type, Test Procedure Text.
  • Justification source columns: Vital Reason 1 to Vital Reason 4.
  • Legacy or workbook values that imply Essential are normalized to Desirable.

Material Handling Workbook Import

Upload one `.xlsx` Chemical Properties Register workbook. The Secretary Workbench will create or reopen the separate Material Handling draft stream, stage the proposed chemical/specification changes, and store material-handling background and justification without altering parameter rows.

Rows are matched primarily by Material Code and staged onto the separate Material Handling draft stream.

Expected Workbook Format

  • One row per specification/material.
  • Required matching columns: Material Code and Proposed New Specification No.. Chemical Name and Specification No. are optional and will fall back to the matched parent specification when omitted.
  • Staged draft fields: proposed Specification No., optional proposed Chemical Name, material properties, storage conditions, and packing fields.
  • Narrative sections are built from workbook data plus the matched parent specification for any omitted current-spec or chemical-name values.
  • Import does not update live master data until the approved publish step.
