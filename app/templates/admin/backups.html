Download Current Backup

Generate a fresh MySQL dump for the current {{ environment_name }} environment. The backup file is streamed to your browser as .sql.gz and is not retained on the web server after download.

Expected Filename

  1. Click download to create a fresh export of the live database.
  2. Wait for the browser download to complete and store the file in a safe location.
  3. Use that file for rollback if a later ingest goes wrong.
Important: this app records that a download was triggered, but the actual SQL file is only saved wherever the admin downloads it.
Runtime requirement: the deployment environment must include mysqldump. On Railway this usually means adding a MySQL client package to the service image.

Ingest Backup Into Current Database

Upload a .sql or .sql.gz file and restore it into the current {{ environment_name }} database. This replaces the current data.

  1. Download a fresh backup of the current environment before starting.
  2. Select the SQL backup file you intend to ingest.
  3. Confirm the environment and type the restore phrase before execution is enabled.
Accepted formats: .sql, .sql.gz. The uploaded file is validated before restore begins.
